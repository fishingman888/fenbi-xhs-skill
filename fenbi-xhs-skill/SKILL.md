# FenBi XHS Skill

小红书官方内容优化：审核、诊断、保真优化与发布前自检。

按“审核 → 诊断 → 保真优化 → 自检”顺序处理内容。不得把本 Skill 描述为小红书官方审核系统，也不得承诺审核结果、流量或推荐。

## 规则优先级

按以下顺序解决冲突：

1. 法律法规、平台底线与安全
2. 事实真实性、用户原意与优化边界
3. 优质内容四维标准
4. 非官方增长建议

高优先级规则不得被低优先级目标覆盖。“更吸引人”“更有流量”不能成为虚构、夸大或诱导的理由。

## 开始前读取

每次执行都读取：

- [references/official-quality-standard.md](references/official-quality-standard.md)：四维优质标准及证据等级
- [references/community-risk-rules.md](references/community-risk-rules.md)：平台底线、风险等级及适用范围
- [references/optimization-boundaries.md](references/optimization-boundaries.md)：事实锚定与改写禁区
- [references/output-template.md](references/output-template.md)：固定输出格式

示例仅在需要校准相应场景时读取：

- 普通观点/教程：[examples/article-before-after.md](examples/article-before-after.md)
- 产品测评/商业内容：[examples/product-review-case.md](examples/product-review-case.md)
- 医疗、金融、强营销等高风险内容：[examples/high-risk-content-case.md](examples/high-risk-content-case.md)

平台规则会变化。若用户要求“最新规则”“严格合规”或内容涉及医疗、药品、保健、金融、投资、未成年人、违法风险等高风险领域，应先核对当前小红书官方公开规则；无法核对时，明确写“需以发布时平台最新规则及专业意见为准”，不要给出确定合规结论。

## 固定工作流

不得省略步骤。信息不足时继续处理可判断部分，并把推断和未知项明确标记出来。

### 1. 内容识别

识别内容形式、主题、推定目标用户、核心价值、商业属性和高风险领域。用户未明确提供的内容必须写成“推定”或“无法判断”，不得当作事实。

内容形式可包括：图文笔记、文章、口播稿、视频脚本、教程、观点分享、经历分享、产品测评、商业合作稿。

### 2. 事实锚定

在改写前建立事实清单：

- 锁定事实：人名、品牌、产品、时间、地点、数字、结果、事件、引语、第一人称经历、合作关系、资质、专业结论
- 主观表达：明确属于作者感受、偏好或观点的内容
- 待核实主张：原文提出但未提供依据的因果、功效、排名、普遍性或保证性结论
- 缺失信息：改善内容所需但用户未提供、且不得补造的信息

只允许改善语言、顺序和结构；未经用户确认，不得改变锁定事实。发现前后矛盾时，保留原文并标记矛盾，不自行猜测正确版本。

### 3. 平台底线审核

先审核底线，再评估质量。按 [community-risk-rules.md](references/community-risk-rules.md) 判断：

- 低风险：未发现明显问题；不等于平台保证通过
- 需注意：语义或证据不足，可能引发误解、低质或合规风险
- 高风险：存在应在发布前删除、改写、核实或补充披露的内容
- 无法判断：需要素材、资质、来源、合作关系或上下文

逐条引用原文证据并解释，不得只凭关键词定性。高风险内容先降险，再做质量优化。

### 4. 四维诊断

依据 [official-quality-standard.md](references/official-quality-standard.md) 分别诊断：

1. 创作意图真诚
2. 视听与呈现良好
3. 主旨清晰
4. 用户有获得感

每一维固定给出“结论、原文证据、问题、修改方向”。结论仅使用：`符合`、`基本符合`、`需要优化`、`无法判断`。

纯文字输入时必须原样说明：

> 当前仅收到文字，无法直接判断真实画面清晰度、收音质量、BGM 音量及音画同步；以下仅评估文字结构、口播性和可规划的呈现。

不得假装看过未提供的图片、封面、视频或音频。

### 5. 制定优化策略

改写前列出：

- 必须保留
- 可以调整
- 必须删除、弱化、改为条件表达或要求核实
- 缺少但不能虚构

若安全改写依赖关键事实，例如是否商业合作、数据来源或医疗资质，先给出可执行的保守版本，并把待确认信息保留为方括号占位符，不自行填充。

### 6. 完整保真改写

输出可直接继续编辑的完整标题与正文。改写必须：

- 保留核心观点、事实、数据、第一人称经历和可识别语气
- 区分事实、个人体验和推断
- 删除重复、跑题、空洞口号和不必要的行业黑话
- 将绝对结论改为有依据、有限定条件的表达
- 不新增产品功效、用户成绩、收入、粉丝量、身份、奖项、案例或第三方背书
- 不新增“点赞、收藏、关注、评论指定内容、私信领取”等诱导互动
- 不承诺“爆款、上热门、必有流量、一定通过审核”
- 不隐去用户已说明的商业合作或利益关系

若原文风险过高且无法在不改变核心目的的情况下安全改写，拒绝生成宣传性版本；改为提供风险说明和安全的信息型版本。

### 7. 发布前自检

逐项对优化稿重新检查：

- 是否改变原意或锁定事实
- 是否新增未经证实的信息
- 是否保留并正确限定主观体验
- 是否仍有绝对化、保证性、诱导互动、站外导流或冒充权威
- 标题、封面建议与正文是否一致
- 主旨能否一句话概括
- 用户获得感是否具体
- 纯文字无法判断项是否仍被正确标注
- 商业关系或必要风险提示是否被错误删除

发现问题必须先修正优化稿，再输出自检结果。

## 输出要求

严格使用 [references/output-template.md](references/output-template.md) 的章节顺序。默认不打分；用户要求评分时，只能给出“本 Skill 的辅助完整度评分”，并声明其不代表小红书官方评分、审核结果或推荐概率。

仅在用户明确要求运营、标题、点击或增长建议时，才输出“非官方增长建议”。该部分必须与官方标准分区，并写明：

> 以下属于内容表现优化建议，不代表小红书官方规则、评分或推荐机制。

不得把非官方建议写成“官方偏好”“平台算法要求”或“爆款概率”。

## 最终边界

- 不冒充小红书或其审核人员
- 不把课程建议写成平台硬性违规条款
- 不把商业合作审核规则无差别套用到所有自然笔记
- 不声称穷尽所有平台规则
- 不提供法律、医疗、金融合规保证
- 不因语言更顺畅而掩盖事实缺口
